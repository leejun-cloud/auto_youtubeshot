# 매뉴얼 — YouTube Shots Creator 배포·운영 가이드

이 문서는 **설치파일을 어떻게 만들고 배포하는가**에 집중한다. 앱 사용법은 `dist-win/README.md` 참고.

## 한 줄 소개

Gemini + Remotion 기반 로컬 웹앱(숏폼/릴스 자동 제작기). 사용자는 설치파일을 더블클릭해
설치하고, 첫 실행 시 자동으로 의존성을 받아 브라우저(`localhost`)로 작업한다.

## 산출물 종류

| 방식 | 명령 | 결과물 | 사용자 경험 |
|---|---|---|---|
| **설치파일(권장)** | `node scripts/build-installers.mjs all` | `dist-installers/YouTubeShots-Setup-1.0.0.exe`, `YouTubeShots-1.0.0-arm64.dmg` | 더블클릭 설치 → 시작메뉴/Applications 등록 |
| 포터블 zip(기존) | `npm run build:all` | `youtube-shots-windows.zip`, `youtube-shots-mac.zip` | 압축 풀고 `start.bat`/`.app` 실행 |

## 설치파일 만들기 (Mac에서 둘 다 빌드 가능)

```bash
# 1) 도구 (최초 1회)
brew install nsis create-dmg

# 2) 재료: 포터블 런타임 포함 배포 폴더 준비 (없으면)
npm run build:win     # → dist-win/  (포터블 Node + 소스)
npm run build:mac     # → dist-mac/  (universal Node + 소스)

# 3) 설치파일 포장
node scripts/build-installers.mjs all      # win + mac
#   node scripts/build-installers.mjs mac  # dmg 만
#   node scripts/build-installers.mjs win  # exe 만
```

결과물은 `dist-installers/` 에 생성된다.

## 각 설치파일 동작 방식 (왜 오류가 없는가)

- **Windows `Setup.exe`(NSIS)**: `dist-win` 전체(포터블 Node 포함)를
  `%LOCALAPPDATA%\YouTubeShots` 에 설치 → **관리자 권한 불필요, 쓰기 가능** →
  첫 실행 시 `npm install` 이 정상 동작. 시작메뉴/바탕화면 바로가기 + 제어판 삭제 등록.
- **macOS `.dmg`(자체포함 `.app`)**: 앱소스+런타임을 `PhotoReels.app` **내부**에 넣고,
  첫 실행 시 `~/Library/Application Support/PhotoReels` 로 복사한 뒤 거기서 구동 →
  `.app` 하나만 드래그 설치해도 깨지지 않음. (기존 zip 방식은 형제 폴더 의존이라 dmg에 부적합했음)

## 사용자 안내 (배포 시 첨부)

- **Windows**: 설치 시 "Windows의 PC 보호" 뜨면 **추가 정보 → 실행** (미서명이라 1회 경고).
- **macOS**: 첫 실행은 **우클릭 → 열기**. "손상됨" 뜨면
  `xattr -dr com.apple.quarantine /Applications/PhotoReels.app`.
  이 dmg는 **Apple Silicon(arm64)** 대상.

## 필수 환경변수 / 시크릿

- 빌드 자체엔 시크릿 불필요(무서명).
- 앱 **실행 시** 사용자가 화면에서 입력: Gemini API 키(필수), Pixabay/Pexels(선택).
  → 사용자 브라우저에 저장되며 저장소/빌드에는 포함되지 않는다.

## 운영 팁

- 버전은 `package.json` 의 `version` 을 따라 파일명이 정해진다.
- dmg가 밋밋하면 `create-dmg` 설치 확인(미설치 시 `hdiutil` 폴백).
- 설치파일이 큰 이유: 포터블 Node 런타임(win ~112MB, mac universal ~140MB)을 번들하기 때문.
